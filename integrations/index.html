Which RemoteDebug bridges/polyfils are available?

We have started the work on providing bridges/polyfils for the major browsers. It's still early days.

  • remotedebug-firefox-bridge

    Mozilla Firefox to Chrome Remote debugging protocol bridge.

  • ios-webkit-debug-proxy

    WebKit Remote Debugging Protocol to Safari Remote Web inspector bridge.

  • bugger

    Node script debugger to Chrome Remote debugging protocol (node-inspector pre-strongloop fork, to stay compatible)

  • node-inspector

    Node script debugger to Chrome Remote debugging protocol (newer version might be in compatible)

  • remotedebug-ie-bridge

    Not yet. Is this your new open source project? Make it happen!

Tools, editors and integrations that work with RemoteDebug

The RemoteDebug protocol (in form of the Chrome Remote Debugging protocol), is already integrated in several tools and editors. Going forward these integrations should be able to work with other browsers, if we suceed with RemoteDebug initiative.

Clients:

  • chrome-remote-interface

    Node client for Chrome Remote Debugging

Editors:

  • Lighttable

    Light Table is a new interactive IDE that lets you modify running programs and embed anything.

  • Sublime Web Inspector

    Debug Javascript right in the Sublime Text editor.

  • Adobe Brackets

    Brackets is an open source code editor for web designers and front-end developers.

  • vim-chrome

    Vim meets Chrome. Experiments at driving a remotely running Chrome instance from Vim.

Tools:

  • Selenium WebDriver for Chrome

    The Selenium WebDriver implementation for Chrome that users Chrome debugging protocol.

  • Chromium Telemetry

    Telemetry is a performance testing framework used by the Chromium project.

  • crconsole

    Remote JavaScript console for Chrome/Webkit.
